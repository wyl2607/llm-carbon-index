# 公正性与可比性

本文档概述了在比较不同模型类别、提供商和来源时，确保 LLM Carbon Index 保持公正且学术严谨所遵循的原则。

## 1. 开源 vs. 闭源模型不对称

开源权重模型与专有“闭源”模型之间存在根本的信息差距。

- **开源模型**：能耗强度来自直接测量（例如 AI Energy Score [E-METHOD]）或已知参数计数和架构。
- **闭源模型**：能耗强度使用 EcoLogits 类假设建模 [E-METHOD, A3]，涉及更宽的不确定性区间以反映未披露的参数计数、硬件和设施效率。

**公正性规则：**
- **使用范围而非中值**：所有跨模型比较**必须**使用完整的 `{low, mid, high}` 范围。禁止按单一中值进行排名或排序，因为这会产生虚假精度。
- **故意宽度**：闭源模型的范围故意更宽以反映其不透明性。
- **中值排名限制**：若闭源模型与开源模型的范围显著重叠，则绝不可仅基于中值比较将闭源模型排在“更好”位置。

## 2. 分词器不可比性 (L-TOKENIZER)

Token 计数由每个模型的专有分词器提供。由于不同分词器将相同文本切分成不同数量的 token，“总 token 数”的简单求和不是工作负载的苹果对苹果比较。

- **更公平的轴**：为缓解此问题，指数将 **每输出 token 的 CO2** 推广为主要效率指标，因为输出 token 是推理中能耗最密集的阶段。
- **警示标签**：无论何处出现跨模型 token 总和或每 token 效率指标，均必须可见 `L-TOKENIZER` 警示，以告知读者这些数字相对于每个模型特定分词器而言是相对值。

## 3. 来源中立性

项目保持严格来源中立。无论模型的 `origin`（例如 CN、US、EU）如何，估算方法学对所有模型均相同。

- **一致方法**：相同的基于物理的估算链（Token → 能量 → PUE → 碳）应用于所有行。
- **区域因子**：计算中唯一与来源关联的差异是假设服务区域的**来源电网因子** [A3]。具有 `cn-north` 来源的模型将使用该区域的电网因子 [C-GRID-CN-NORTH-537]，正如 `us-east` 模型使用其各自因子 [C-GRID-US-EAST-380]。不根据地缘政治来源施加任意“惩罚”或“奖励”。

## 4. 流量加权

OpenRouter 流量严重偏斜；少量热门模型往往主导总足迹。

- **按 token 加权的总数**：每日总数按 `total_tokens` 进行 token 加权，以反映当日实际使用的环境影响。
- **未加权伴随物**：为确保图片不单纯是来自一两个高流量模型的工件，指数报告总数的**未加权伴随物**。这允许研究者看到独立于当前市场流行度的已建模模型库的平均足迹。

## 原则总结

通过遵守这些规则——优先使用范围而非点数、承认分词器差异、在不同来源间保持方法一致性，并提供未加权比较——LLM Carbon Index 确保其排名是对现有最佳证据的公正表示。
